# 메모

## 뷰 로더

[뷰 로더 가이드](https://vue-loader.vuejs.org/guide/#manual-setup)

```
npm install -D vue-loader vue-template-compiler
```

## 바벨로더, 스타일로더 설치하기

뷰 로더는 컴포넌트는 해석이 가능하지만 자바스크립트나 스타일에 관한 해석이 필요한 로더를 설치해야 한다.

```
npm install -D babel-loader vue-style-loader css-loader
```

### 바벨 (Babel)

- ES6 이상의 문법의 코드를 ES5 이하 버전으로 변환하여 사용. 프로그램의 호환성을 높임.
- @babel/core: 바벨이 실질적으로 동작하는 모듈
- @babel/preset-env: 바벨의 지원 스펙을 지정. (일일이 명시하기가 어려우니까 명시해야 할 기본적인 내용을 세팅해 둔 패키지)
- babel-loader: 웹팩 지원을 위해 사용하는 패키지

```
npm install -D @babel/core @babel/preset-env
```

- @babel/polyfill: 구형 브라우저 지원을 위해 설치
  - 바벨을 이용하지만 구형 브라우저에서 지원되지 않는 기능을 이용할 수 있게 해 줌

```
npm install @babel/polyfill
```

### html-webpack-plugin

```
npm i -D html-webpack-plugin
```

### copy-webpack-plugin

ico, png 파일을 dist폴더에 넣어줄 수 있도록 세팅하기 위해 필요한 모듈

```
npm i -D copy-webpack-plugin
```

### clean-webpack-plugin

빌드할 때 파일이 삭제되지 않고 덮어씌우기 때문에 이미지 리소스의 경로명을 바꾸는 등의 변경사항이 발생한 경우에는 이전 경로의 리소스를 지워줘야 하는 번거로움이 있다. 따라서 빌드할 때마다 dist폴더를 비워주는 모듈을 설치

```
npm i -D clean-webpack-plugin
```

## webpack-dev-server

일일이 수정사항을 반영하고 빌드하기에는 시간이 오래 걸린다. 실시간으로 변경사항을 확인하고 싶다면 webpack-dev-server을 설치하여 확인할 수 있다.

그러나 전역적으로 설치한 것이 아니므로 package.json의 scripts에 정의하여 사용하도록 한다.

```
npm i -D webpack-dev-server
```

사용법은 다음과 같다.

```
webpack-dev-server --mode devleopment
```

## webpack-merge

개발용, 배포용 겹치는 옵션과 다른 옵션을 분기할 수 있음.
