## ✔ 강의 선택 동기

### Vue를 배우기 시작한 동기

개인적으로 프런트 엔드 개발자로 취업하기 위해서 1년간 `React`를 공부해 왔습니다.
그러나 가끔 면접을 보면서 React가 아닌 `Vue`를 주로 사용하는 회사를 종종 봐왔습니다.
주로 이런 회사들은 `Vue, React, Angular` 중 하나 이상을 개발할 수 있는 사람들을 뽑는다고 채용 공고를 올립니다.
당시에는 하나의 분야 중에서도 하나의 우물만 파는 것이 얼마나 힘든지를 잘 알기 때문에 `Vue`까지 배우기에는 벅차다는 느낌이 들었습니다. 그런데 회사에서 Vue, React, Angular 중 하나만 알아도 채용한다는 조건을 내건 것을 보면서 어쩌면 `React`에 익숙해졌으니까 `Vue`를 새로 배우며 개발하는 것 또한 가능하지 않을까 하는 생각이 들었습니다. 그래서 `Vue`를 사용하는 회사에 입사를 확정을 짓게 되면서 Vue.js 강의를 고르기 시작했습니다.

### 해당 강좌를 고른 이유

생각보다 Vue는 React보다는 배우기가 쉽고 접근성이 낫다는 생각이 들었습니다. 일단 API 문서를 보여주는 인터페이스부터 깔끔하고 한국어로 지원하는 API가 너무 인상적이었습니다. 더더군다나 개인적으로 느끼기에 `Vue`를 가르치는 한국인 강사진이 React보다는 상대적으로 많은 것 같다고 느꼈습니다. 저는 그중 [`박영웅`](https://github.com/ParkYoungWoong) 선생님을 강좌를 선택했습니다. 이전에 개발하면서 [CSS flex layout](https://heropy.blog/2018/11/24/css-flexible-box/)에 관해 깊게 공부를 해야 했는데 박영웅 선생님이 포스팅하시는 [Heropy Tech 블로그](https://heropy.blog)를 검색 엔진을 통해 우연히 보게 됐던 때가 있습니다. 정말 세세하게 레이아웃에 관한 설명이 나와 있어서 도움이 되었기 때문입니다. 이 선생님께 배운다면 뭔가 더 쉽게 Vue를 배울 수 있을 것 같아서 이 강좌를 고르게 되었습니다.

### 💯 학습 의도

1. 이제 React뿐 아니라 Vue도 친근하게 사용하는 도구로 만들고 싶습니다.
2. 현재 입사한 회사에서 Vue를 주력으로 사용하고 있습니다.
   개발하면서 Vue가 걸림돌이 되지 않기를 바라는 마음에서 선행학습을 수행하고자 했습니다. (2020. 8. 24. ~)
3. 공부하면서 React로 프로젝트를 작성하는 스타일과 Vue를 이용하여 프로젝트를 작성하는 스타일의 차이점을 경험해 보고 싶습니다.
    - 주로 두드러지는 차이점
        1. React는 `JSX`로 컴포넌트를 구성합니다. JSX도 Javascript가 품은 XML문법이기 떄문에 좀 더 자바스크립트답게 사용할 수 있습니다. 예를 들어 주로 CSS 파일을 import하여 불러오기보다는 `Styled-Components` 라이브러리를 이용하여 컴포넌트 객체에 Tagged Template Literal 문법을 이용하여 스타일링합니다. 그러나 Vue의 경우에는 `template`, `script`, `CSS/SCSS` 부분을 나누어 하나의 컴포넌트를 구성합니다. template 영역에는 완전한 html 문법을, CSS 영역에는 완전히 CSS, SCSS의 문법을 이용해야 하므로 프로젝트를 작성해 나가는 스타일에 차이가 있습니다.
        2. 이전에는 React가 클래스 문법을 이용할 때는 라이프 사이클을 작성해야 했습니다. 어쩌면 클래스를 선언하는 것만 제외한다면 지금의 Vue의 구조와 유사해 보입니다. Vue는 그렇게 라이프 사이클(ex: create, mount, update, destroy 등)을 이용하지만, 함수형 컴포넌트로 구성되는 React의 경우에는 `useState`, `useEffect` 등의 함수를 적시에 활용해야 합니다.
        3. React에서는 객체의 불변성을 지키려고 노력합니다. React의 상태에서는 객체를 수정할 때는 직접 수정하지 않고 `...object`와 같이 기존 객체를 복사하여 새로운 객체를 만드는 것이 일반적입니다. 객체를 참조하는 값이 바뀌었다는 것을 리액트가 감지한 다음에야 컴포넌트를 업데이트하기 때문입니다. `Redux`나 `Context API`에서도 마찬가지입니다. 하지만 현업에서 느낀 `Vue`의 경우에는 이러한 것에 특별히 제약을 두지 않는 느낌입니다.
